# Contributing docs

Docs for people working **on** Range, not just using it. Start with
[`../../CONTRIBUTING.md`](../../CONTRIBUTING.md) at the repo root —
that's the entry point for new contributors.

The docs in this folder go deeper:

| Doc | When to read it |
|---|---|
| [`architecture.md`](architecture.md) | Read first. Codebase tour — server modules, frontend layout, the data flow for one scenario run. |
| [`product_spec.md`](product_spec.md) | What we're building and why. The v0.5 spec is currently authoritative; v0.7 in progress. |
| [`positioning.md`](positioning.md) | Hard product rules (e.g., Range stays NVIDIA-independent; integrations only via public repos). Read this before proposing any feature that pulls in a new substrate or vendor. |
| [`direction/`](direction/) | Time-stamped strategy snapshots. Useful historical context; not authoritative. |

Anything that's no longer authoritative lives in [`../archive/`](../archive/).
